About me

Hi! I'm Carolina, and during the last four months I've been studying hard to start my career in front-end development. My goal is to know enough about the recent technologies to get an internship where I'll be able to grow as a professional and push start my career in the field.

I currently believe that there are three main rules during the process of building a website or web application:

INTERACTIVITY.

Enabling a positive interaction with possible clients can be a game changer. I think that there are two sides of interactivity: the bad side, when a pop-up appears, begging the user to register or like their facebook page in order to view the content, and the good side, when the website can somehow connect to the user and make them feel valuable and important.

SMART POSITIONING.

Every element should have a reason to be exactly where it is. As the web is growing fast and renewing itself every day, it doesn't matter if it is a color palette, a big column list or a different typography: if it adds up to allow a positive experience, then it's the job of a good front-end developer to code it and make the end result as light, smooth and natural as possible.

FUNCTIONALITY.

It needs to work. Correctly. And should be developed in a way that when the user does something unexpected, it wont spam the screen with scary messages and errors. Yet it should present the user with a complete experience: whether if it is just a small form on a page or a big e-commerce application with complex processes and components, it should not be left halfway done.
